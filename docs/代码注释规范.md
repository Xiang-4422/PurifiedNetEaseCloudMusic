# 代码注释规范

## 1. 目标

注释只服务一件事：帮助后来者理解代码职责、约束和取舍。

默认用中文。库名、协议名、平台术语和外部系统名可以保留英文。

## 2. 基本原则

- public 成员必须有 Dart 文档注释。
- 注释优先解释“为什么”，其次说明职责、输入输出和副作用。
- 方法体内部少写注释，只写特殊约束、非直观实现和误用风险。
- 不确定真实原因时不要猜。
- 禁止机械批量生成注释。

## 3. 必须注释

以下 public 声明必须使用 `///`：

- class、enum、extension、mixin、typedef。
- 方法、函数、构造函数、getter、setter。
- 字段、属性、顶层变量、常量。
- domain entity、repository、service、data source、DAO、controller、adapter 的 public 成员。

private 成员只有在存在特殊约束、workaround、并发、缓存、生命周期、播放器联动等风险时才补注释。

## 4. 禁止注释

不要写只复述代码的注释，例如：

- 初始化某对象。
- 设置某字段。
- 调用某方法。
- 判断是否为空。
- 循环列表。
- 把 A 赋值给 B。

如果注释删掉“为什么”后只剩行为描述，通常应该删除或改写。

## 5. 推荐写法

声明级注释：

```dart
/// 保存播放恢复状态，供应用冷启动后恢复上次队列和进度。
class PlaybackRestoreState {}
```

局部注释：

```dart
// just_audio 的 play() 会在播放结束后才完成，这里不能 await。
unawaited(_player.play());
```

## 6. 临时方案

临时方案必须写清：

- 为什么现在只能这样。
- 在规避什么问题。
- 未来什么条件下可以替换。

禁止只写：

```text
临时处理
hack
兼容问题
```

## 7. 已知注释相关工程问题

补 public API 注释时发现的仍需关注事项：

- Drift 表定义仍偏集中。
- 图片主色缓存仍依赖 Hive 全局入口。
- 部分领域实体仍承担 JSON 编解码。
- `PlaybackQueueItem` 仍保留播放适配字段。
- 用户作用域本地数据源接口偏宽。
- 网易云远程访问入口存在能力重叠。
- 网易云 SDK DTO 文件体量偏大。

这些问题进入后续重构路线，不要求在注释工作中一次性解决。
