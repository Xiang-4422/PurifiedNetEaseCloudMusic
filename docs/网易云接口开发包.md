# 网易云接口开发包

## 1. 目标

把 `NeteaseCloudMusicApiEnhanced/api-enhanced` 复刻为可长期维护的 Dart SDK，并支持后续跟随上游刷新。

SDK 层只负责：

```text
接收 query -> 构造网易云内部请求 -> 选择加密方式 -> 发送请求 -> 返回原始响应
```

不负责 App 缓存、播放队列、UI、本地数据库和领域实体。

## 2. 上游来源

- 上游仓库：https://github.com/NeteaseCloudMusicApiEnhanced/api-enhanced
- 本地 submodule：`third_party/api-enhanced`
- Dart SDK：`packages/netease_music_api`

## 3. 刷新流程

```bash
git submodule update --remote third_party/api-enhanced
git diff --submodule

node packages/netease_music_api/tool/generate_api_enhanced_modules.js

flutter test packages/netease_music_api/test/api_enhanced_raw_test.dart
flutter test test/architecture
flutter analyze
```

首次拉取包含 submodule 的仓库后执行：

```bash
git submodule update --init --recursive
```

## 4. 生成器

生成器：

```text
packages/netease_music_api/tool/generate_api_enhanced_modules.js
```

生成产物：

```text
packages/netease_music_api/lib/src/generated/api_enhanced_modules.g.dart
packages/netease_music_api/lib/src/endpoints/raw/api_enhanced_raw_methods.g.dart
```

当前规则：

- 覆盖上游 `module/*.js`。
- 无 request path 的模块标为 `special`。
- 暂不支持的 crypto 标为 `special`。
- 生成器会格式化 Dart，并且只在内容变化时写回。

自动支持的 crypto：

```text
weapi
eapi
linuxapi
api
xeapi
```

`xeapi` 需要先完成 `register_xeapikey`，SDK 会持久化公钥状态并在请求响应后更新 xeapi session。

## 6. 当前差异

已实现或已有手写入口：

- `api`
- `audio_match`
- `avatar_upload`
- `cloud`
- `cloud_upload_token`
- `cloud_upload_complete`
- `eapi_decrypt`
- `decrypt` 的 eapi-compatible 分支
- `inner_version`
- `login_qr_create`
- `playlist_cover_update`
- `register_xeapikey`
- `register_anonimous`
- `related_playlist`
- `song_url_v1` 的 xeapi 基础请求
- `vip_sign_history`
- `vip_tasks_v1`
- `voice_upload`

运行时 option 当前状态：

- `realIP` 会写入 `X-Real-IP` 和 `X-Forwarded-For`。
- `randomCNIP` 会在 Dart raw metadata 阶段生成请求 IP，显式 `realIP` 优先。
- `ua` 会映射为原始 User-Agent。
- `checkToken` 会写入 eapi header。
- `proxy` 支持 HTTP/HTTPS 代理 URL，并在 Dart native `HttpClient` 层生效。
- `domain`、`cookie` 已进入 request options。
- 已建立 Node oracle fixture，覆盖 `album`、`search`、`cloudsearch`、`song_detail`、`song_url`、`lyric_new`、`playlist_detail`、`playlist_subscribe`、`playlist_tracks`、`user_playlist`、`like`、`personalized`、`recommend_songs`、`search_hot_detail` 等常用模块，用于对比上游模块传给 `request` 的 URI、data 和 options。

仍未完整复刻：

- `song_url_v1` 的 unblock 分支依赖 `unblockmusic-utils`。
- `song_url_match` 依赖 `unblockmusic-utils`。
- `song_url_ncmget` 当前返回空 data。
- `decrypt` 尚未完整覆盖 `weapi`、`linuxapi`、`xeapi`。
- `proxy` 暂不支持 PAC 和带认证的代理。
- FLAC/source order 仍需结合 unblock 替代链路评估。
- Node oracle 仍需继续扩大到更多 normal module 和 special module。

## 7. 测试要求

raw manifest 测试必须覆盖：

- manifest 覆盖所有上游 module。
- generated method name 唯一。
- normal module 能构建 request metadata。
- normal module 有 path。
- normal module crypto 受支持。
- special module 清单显式固定。
- `xeapi` module 必须 special。
- pathless module 必须 special。
- unsupported crypto 不能静默 fallback。
- `randomCNIP` 必须生成可用 IPv4，且不能覆盖显式 `realIP`。
- `proxy` 必须能映射为 Dart native proxy rule，不支持的代理形式必须显式报错。
- Node oracle fixture 必须验证 Dart metadata 与上游 Node request metadata 一致。

当前验证结果：

```text

architecture tests: 必须通过
full test suite: 必须通过
flutter analyze: No issues found
git diff --check: passed
```
