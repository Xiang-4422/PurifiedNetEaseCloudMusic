# 重构路线

## 1. 目标

后续重构目标是把当前应用收敛成一个自用、简约、快速、稳定的音乐播放器。

不做：

- 不推倒重来。
- 不迁移状态管理框架。
- 不重写播放器。
- 不把 App 缓存逻辑放进 SDK。
- 不让 UI 直接消费 SDK raw JSON。

## 2. 已完成方向

- 项目已从固定 application/usecase/port 分层收敛为 feature-first。
- 已删除大量只转发的薄层。
- 歌单详情已走本地优先。
- 自有业务 snapshot 持久化已移除。
- Drift 已成为业务事实来源。
- Hive 范围已收窄。
- `packages/netease_music_api` 已独立成 SDK 包。
- bootstrap 已从单个大文件拆成 UI、SDK、数据、播放、展示适配器、feature/controller 装配文件。
- `UserScopedDataSource` 已拆出用户资料、曲目列表、歌单列表、订阅状态、电台、同步标记等窄接口。
- `DriftUserScopedDataSource` 已退化为聚合器，真实表访问已拆到窄 Drift data source。
- 用户作用域窄 Drift data source 已只保留 DAO 委托，用户歌单列表和电台缓存表访问已下沉到 DAO。
- Drift 表定义已从 `drift_database.dart` 拆到 `schema/drift_tables.dart`。

## 3. 架构整理

优先级：

1. 收紧 SDK 与 App 边界，确认 SDK 调用只在网易云 remote data source。
2. 拆分 Drift 表、DAO 和 data source。
3. 继续拆分 `drift_database.dart` 里的迁移和索引。
4. 收紧 `PlaybackQueueItem`，避免携带 adapter、JSON、SDK 细节。
5. 继续收紧 bootstrap 内部各装配文件的依赖方向。

## 4. 登录和账号

目标边界：

- SDK 管 cookie、token refresh、request session。
- App 管当前用户、账号作用域、本地缓存归属。

需要梳理：

- 冷启动恢复顺序。
- cookie 失效和刷新失败。
- 登出和切换账号。
- `user_*` 表和刷新 marker 的 `user_id` 隔离。

## 5. 缓存策略

需要统一：

```text
歌曲详情      Drift 长期保存，远程刷新补全
歌单          元信息和曲目关系分开缓存
歌词          缺失拉取，成功后长期保存
封面          文件缓存和取色缓存分层
播放 URL      短 TTL 或内存缓存
喜欢歌曲      账号作用域
用户歌单      账号作用域
云盘          账号作用域
评论          短期缓存或后续建专表
搜索热词      短 TTL
```

## 6. 播放和下载

播放方向：

- 本地资源优先。
- 远程 URL 过期后刷新。
- 远程失败时支持重试或降级。
- 下一首预取不阻塞当前播放。
- 恢复状态只保存 restore/session 语义。

下载方向：

- `download_tasks` 描述下载过程。
- `local_resource_entries` 描述最终可用资源。
- 播放只信任本地资源索引，不直接信任下载任务状态。

## 7. UI 方向

定位：自用、简约、纯粹听歌。

核心页面：

- 首页：最近播放、每日推荐、常用歌单。
- 资料库：喜欢、歌单、本地音乐、下载、云盘。
- 搜索：搜索框、热搜、历史、结果。
- 播放页：封面、歌曲信息、进度、歌词、队列、音质。
- 歌单页：封面、标题、操作区、歌曲列表。
- 设置页：账号、音质、缓存、下载、外观、调试。

避免：

- 信息流、复杂社交、运营 banner 成为主入口。
- 卡片套卡片。
- 过度模糊或取色影响可读性。
- 动效影响播放控制响应。

## 8. 建议 goal 顺序

1. 收紧 SDK 与 App 边界。
2. 明确登录、cookie、账号状态所有权。
3. 统一核心缓存策略。
4. 重构播放源解析和 URL 缓存。
5. 整理下载和本地资源索引。
6. 收紧 bootstrap 装配边界。
7. 拆分 Drift 本地数据层。
8. 梳理首页、资料库、搜索、播放页、mini player、歌单页。
9. 优化视觉系统和核心动效。
10. 建立启动、歌单打开、搜索、切歌等性能指标。
