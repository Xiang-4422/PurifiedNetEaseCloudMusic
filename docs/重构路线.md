# 重构路线

## 1. 目标

后续重构目标是把当前应用收敛成一个自用、简约、快速、稳定的音乐播放器。

不做：

- 不推倒重来。
- 不迁移状态管理框架。
- 不重写播放器。
- 不把 App 缓存逻辑放进 SDK。
- 不让 UI 直接消费 SDK raw JSON。

## 2. 已完成方向

- 项目已从固定 application/usecase/port 分层收敛为 feature-first。
- 已删除大量只转发的薄层。
- 歌单详情已走本地优先。
- 自有业务 snapshot 持久化已移除。
- Drift 已成为业务事实来源。
- Hive 范围已收窄。
- `packages/netease_music_api` 已独立成 SDK 包。
- bootstrap 已从单个大文件拆成 UI、route、SDK、storage、数据源、repository、数据装配、播放、展示适配器、feature/controller 装配文件。
- 根路由实例、初始路由和登录失效导航已收口到 `route_bootstrap`，App 根组件只承接 `GetMaterialApp.router`。
- 登录态 UI 副作用监听已从 `lib/app.dart` 下沉到 UI widget，App 根组件不再直接导入 auth controller、dispatcher 或 ToastService。
- SDK 初始化和 `NeteaseMusicApi` 实例创建已收口到 `sdk_bootstrap`，数据装配只接收并注入已创建的 API 门面，`data/music_data/sources/netease` 不再保留远程初始化入口。
- `app_bootstrap.dart` 的 import 白名单已由架构测试固定，只能导入子 bootstrap、Flutter binding 和 GetX，避免入口重新直接依赖 data、features、ui 或 SDK 业务文件。
- Drift/Hive 初始化已收口到 `storage_bootstrap`，数据装配只消费已初始化的存储资源。
- 本地 data source、网易云 remote data source、搜索/探索 cache store、本地音乐来源和网易云音乐来源已收口到 `data_source_bootstrap`。
- 核心 repository 和 feature repository 创建/注册已收口到 `repository_bootstrap`，数据装配只保留启动顺序和恢复任务。
- Feature repository 已改为依赖 `data/music_data/music_remote_data_sources.dart` 中性远程契约，网易云 remote 具体实现只在 `data_source_bootstrap` 创建、在 `data/music_data/sources/netease/remote` 内实现。
- `UserScopedDataSource` 已拆出用户资料、曲目列表、歌单列表、订阅状态、电台、同步标记等窄接口。
- `DriftUserScopedDataSource` 已退化为聚合器，真实表访问已拆到窄 Drift data source。
- 用户作用域窄 Drift data source 已只保留 DAO 委托，用户歌单列表和电台缓存表访问已下沉到 DAO。
- 用户作用域旧 `UserDao` 已拆为资料、曲目列表、歌单订阅状态、同步标记等窄 DAO，避免账号作用域表访问重新堆回一个大类。
- Drift 迁移和索引 SQL 已从 `drift_database.dart` 拆到 `schema/drift_schema_maintenance.dart`。
- Drift 表定义已从 `drift_database.dart` 拆出，并按 playback、local_resource、download、cache、library、playlist、user 业务域拆分。
- `PlaybackQueueItem` 已移除 `MediaItem` extras 和缓存 JSON 编解码职责，转换逻辑留在 playback adapter/codec。
- `PlaybackQueueItem.albumId`、`PlaybackQueueItem.artistIds`、`PlaybackQueueItem.sourceType`、`PlaybackQueueItem.localLyricsPath` 和 `PlaybackQueueItem.availability` 已字段化，UI 和服务层不再读取对应 metadata 动态键。
- 播放队列 MediaItem 和缓存 JSON 写出路径只使用显式字段；legacy metadata 只在读取旧缓存时迁移。
- 播放队列 mapper 已改为读取 `Track.resolvedAlbumId` 和 `Track.resolvedArtistIds`，不再直接解析 `Track.metadata` 的旧 id 键。
- 播放队列转 `MediaItem` 的本地封面 `artUri` 已复用本地路径归一规则，远程 URL 或非法本地路径不会被伪装成本地文件 URI。
- 播放队列转 `MediaItem` 的 extras `image` 已优先写入可用本地封面路径，本地路径无效时才回退远程封面 URL。
- 播放队列 mapper、MediaItem adapter 和缓存 codec 已过滤 `mv`、`fee`、`publishTime`、云盘和本地扫描来源等非播放 metadata，播放队列只保留自定义扩展。
- `Track.albumId` 和 `Track.artistIds` 已字段化，本地 DAO 使用专辑列和歌手关系表保存，旧 metadata 键只作为读写迁移输入。
- 登录标记、用户 session、设置项和图片主色缓存已通过 key-value 边界访问 Hive，`CacheBox` 全局入口不再暴露给对应存储实现。
- Hive 中的当前用户缓存只保留 session 语义，`userSessionKey` 继续使用旧 `USER_INFO` 字符串兼容已安装数据；轻量存储 key 使用语义化命名，不再保留 `Sp` 或用户资料业务快照命名。
- 旧 Hive 业务事实 key 已从 `app_cache_keys.dart` 移除，喜欢歌曲、用户歌单、推荐歌曲、下载任务、本地资源索引和播放恢复不再保留轻量存储入口。
- 搜索热词、探索分类和评论等短期 `app_cache_entries` key 已迁到 `AppCacheDataSource` 边界，feature cache store 不再依赖 `data/app_storage` key。
- 用户资料库和首页推荐的账号作用域本地数据加载已加入过期请求丢弃，旧账号异步返回不能覆盖当前账号状态。
- 首页推荐本地启动会按推荐歌单、日推和 FM 分别容错，任一缓存读取失败不会阻断其它首页缓存展示；启动同步标记读取失败按过期处理。
- 喜欢歌曲队列加载会在落状态前复核账号、请求代次和喜欢歌曲 id 快照，旧账号或旧列表的慢返回不能覆盖当前队列。
- 喜欢歌曲队列本地歌曲缓存读取失败时会继续远程拉取，缓存损坏不会阻断“我喜欢”播放入口加载。
- 喜欢歌曲列表远程重载成功后才替换当前列表，远程失败不会清空已展示的喜欢歌曲。
- 喜欢歌曲 id 刷新成功后会同步裁剪已加载的喜欢歌曲可见列表，并按新的 id 顺序保留仍有效的歌曲。
- 用户资料库本地启动会按字段容错，喜欢歌曲 id、用户歌单、我喜欢入口或随机封面读取失败不会阻断同账号其它缓存展示。
- 用户资料库启动刷新已改为远程快照整组成功后再更新可见状态，远程任一路失败时保留当前资料库状态。
- 用户资料库远程刷新成功但歌单为空时会同步清空可见喜欢入口和用户歌单，避免旧账号或旧列表继续残留。
- 用户资料库 controller 会先规范化当前用户 id，空白账号不会被视为已有本地资料库数据；仓库继续拒绝空白用户 id，避免空账号读取或写入资料、喜欢歌曲、歌单、推荐和 FM 缓存，也不会触发对应远程请求。
- 后台登录态校验返回前会复核发起校验时的 `userId`，旧账号校验结果不能覆盖或清空新账号状态。
- 缓存登录恢复完成后会复位登录加载态，避免本地登录恢复成功但登录页仍停留在 loading。
- 后台登录态校验遇到网络异常时保留当前缓存 session，只有服务端明确返回未登录才清空本地账号状态。
- 用户 session 启动恢复已接入 `AuthRepository.hasCachedSession` 守卫，孤立 session 会和孤立登录标记一样被清理，不能绕过登录页恢复当前用户。
- 只有登录标记但没有可用本地 session 时，会直接回退到二维码登录入口，不再凭 SDK cookie 远程恢复当前用户。
- 主动注销时本地账号 session 和登录标记会优先清理，远程 logout 只作为 SDK cookie/request session 清理尝试。
- 主动注销复用 `AuthController` 的登录页副作用通道，清理本地账号后回到登录页，UI 不直接操作登录路由。
- 登录失效展示副作用已由根组件 dispatcher 消费，回登录页路由失败会记录诊断并消费当前 effect，不再泄漏未处理异步错误。
- 用户 session 持久化已串行化并加入代次保护，旧账号慢保存不能覆盖后续清理或新 session，后台持久化失败不会泄漏到全局异步错误通道。
- Auth 缓存恢复、二维码创建、二维码轮询和授权后账号加载已纳入流程代次保护，旧流程慢返回不能在注销或新登录流程后恢复账号状态。
- `AuthController` 已通过 `AuthSessionAccess` 读写当前 App session，真实 `UserSessionController` 绑定只留在 `feature_bootstrap` 装配层。
- `UserLibraryController` 已通过 `UserLibrarySessionAccess` 读取当前账号、等待 session 缓存并监听账号变化，真实 `UserSessionController` 绑定只留在 `feature_bootstrap` 装配层。
- 二维码创建失败和轮询短暂异常会保留可重试状态，登录页不再因为没有二维码 URL 而空白。
- 用户资料页本地资料读取失败时会继续远程加载，缓存损坏不会直接进入错误态。
- 用户资料页有本地资料时后台刷新失败会保留当前资料并挂载错误状态，首次加载失败时才进入纯错误态。
- 用户资料页刷新会丢弃旧代次结果，页面释放后慢返回的资料请求不能继续写入已释放状态。
- 用户资料页 controller 会先规范化工厂注入的当前用户 id，空白账号不会读取资料缓存，也不会触发远程资料请求。
- 歌单页本地初始化缓存读取失败时按空本地数据处理，页面继续拉取远程首屏，缓存损坏不能阻断歌单打开。
- 专辑页和歌手页本地详情缓存读取失败时按无本地详情处理，页面继续远程加载。
- 本地资源索引保存和读取会验证文件真实存在，失效索引不会再进入播放资源包或本地歌曲列表。
- 下载仓库和批量下载规划会先规范化曲目 id，再拒绝空白曲目 id，避免异常队列项创建空白或重复 `download_tasks`，也避免带空格 id 绕过本地资源查询和 active task 判断。
- 播放源解析本地文件和网易云缓存文件前会二次确认文件存在，下载或缓存文件缺失时回退远程 URL，本地导入文件缺失时明确失败。
- 播放源解析发现队列里的本地音频路径缺失时会触发资源索引修剪，避免失效 `local_resource_entries` 在后续播放中反复命中。
- 播放 URL 强制刷新启动时会移除同 key 旧短缓存，刷新结果只允许最新请求写入短缓存，旧的慢请求晚返回时不会覆盖已刷新的可用 URL。
- 播放 URL 强制刷新远程地址前也会先重查本地资源索引，本地导入、正式下载或播放缓存仍然优先于远程刷新。
- 播放 URL 短缓存会拒绝空白曲目 id，并把空白音质键归一为默认音质，避免异常队列项触发无效本地查询或远程请求。
- `MusicDataRepository` 的曲目详情、批量曲目读取、播放 URL、封面、歌词和资源索引入口会拒绝空白曲目 id，批量资源查询保持候选 id 顺序，避免异常队列项继续打到本地库或网易云远程源。
- `MediaItem.extras['url']` 只保存可恢复的本地路径，不再写入远程短效播放 URL，通知栏和恢复边界不会携带过期播放地址。
- 图片展示缓存和临时目录清理会跳过仍被本地资源索引引用的文件，缓存分析也只统计可安全清理的容量。
- 播放恢复队列缓存按条容错，坏缓存项或恢复队列解码失败不会打断启动恢复，也不会把旧歌单上下文带回空队列。
- 播放恢复只有在持久化的当前歌曲仍能命中恢复队列时才恢复播放进度；当前歌曲缺失或进度异常时只恢复队列上下文，避免把旧进度 seek 到另一首歌。
- 播放状态同步器确认切到新歌曲时会同步归零运行态进度和 restore 进度，不再把上一首的旧进度通过独立后台任务写到新歌恢复状态上。
- 播放恢复进度进入持久化边界前会拒绝负数异常值，避免损坏的运行态污染 restore/session 状态。
- 播放恢复状态和进度写入失败时会保留待写候选；后续进度写入会先补写完整 restore state，避免临时存储失败丢失队列和当前歌曲上下文。
- 播放队列确认底层歌曲时会校验歌曲 id 和索引一致，旧切源完成不能把新队列同索引歌曲误标为 confirmed。
- 歌词资源索引读取文件前已归一普通路径和合法 `file://` 路径，避免通过存在性检查后又按原始 URI 读取失败。
- mini player 圆形播放进度已使用统一进度比例 helper，负数、超出时长、未知时长和非有限值都有测试固定。
- mini player 已补充展开播放器和播放/暂停的辅助语义与 Tooltip，收起态控制入口可被稳定识别。
- 底部播放页进度条已复用安全总时长和播放位置 helper，未知、零值、负数和超出总时长的边界都有测试固定。
- 通用图片组件的空图和自定义 replacement 占位尺寸已有 UI 测试固定，加载失败或空路径不会让封面区域收缩。
- 播放队列条目已固定歌手兜底、高亮颜色和辅助语义，当前播放项可被稳定识别。
- 播放页控制按钮已统一辅助语义和触控区域，喜欢、播放/暂停、上一首、下一首和播放模式按钮都有稳定标签。
- 播放页喜欢按钮的状态读取和切换都已收口到 `PlayerController` 播放边界，同一曲目并发切换会合并，避免快速重复点击打出多次喜欢状态请求和队列更新。
- 播放模式按钮会按私人 FM、心动模式、随机播放、单曲循环、不循环、列表循环和分组循环输出当前状态标签。
- 播放模式切换已用显式心动模式上下文替代 `dynamic` Map，缺少启动歌曲时会回退普通列表，避免 UI 命令层靠动态键进入心动状态。
- `PlayerController` 不再直接依赖 `PlaybackQueueStore` 写播放模式，播放模式同步只经 `PlaybackQueueService` 一次落到 restore state，避免 controller 重复持久化。
- 通用列表项已提供标题和副标题合并后的稳定语义标签，搜索、歌单和歌曲列表入口可被辅助功能和测试稳定识别。
- 播放页控制区已提供轻量音质入口，读写都经 `PlayerController` 播放边界转到播放偏好端口，再通过既有播放链路影响后续播放源解析。
- 搜索热词会先展示本地缓存再后台刷新；热词刷新和搜索结果加载都会丢弃旧请求返回，慢返回的旧关键词或旧搜索结果不能覆盖当前搜索状态。
- 搜索热词缓存读取或新鲜度判断失败时会继续请求远程热词，短期缓存异常不能阻断搜索面板入口。
- 搜索同一关键词强制刷新时会保留当前结果，刷新失败只挂载错误状态，不清空用户正在查看的歌曲、歌单、专辑和歌手结果。
- 搜索结果复用已纳入账号和喜欢歌曲 id 上下文，账号切换或喜欢态变化后同一关键词会重新加载，旧账号结果不会继续复用。
- 搜索 controller 和仓库会先 trim 当前用户 id，空白账号不会读取用户歌单缓存，也不会因为前后空格导致同一账号重复搜索。
- 顶部搜索 Widget 只提交关键词，当前用户和喜欢歌曲上下文由 `SearchPanelController` 注入 provider 读取，避免 UI 直接拼搜索请求上下文。
- `SearchPanelController` 构造时必须显式接收当前用户和喜欢歌曲 provider，不能用空账号上下文兜底绕过账号作用域。
- 顶部搜索面板接收首页壳层注入的 shell、搜索和播放控制器，面板内部不再读取全局容器。
- 搜索面板释放后会取消当前生命周期挂起的热词和搜索请求，慢返回不能继续写入状态；重新挂载时会对当前搜索词补跑一次搜索，避免同关键词因为没有文本变化而卡在旧加载态。
- 评论列表和楼层回复已使用 Drift `app_cache_entries` 做短 TTL 缓存，缓存新鲜时优先返回，远程刷新失败时回退缓存；当前页面已有评论时，刷新失败不会清空可见列表。
- 评论列表和楼层回复的缓存读取、新鲜度判断或缓存写入失败时不会阻断远程结果，远程失败时只尽力回退可用旧缓存。
- 评论 controller factory 已由播放面板组合层注入评论页，并继续传给评论列表、评论项和楼层回复弹层；评论子组件内部不再读取全局容器。
- 评论列表和楼层回复刷新会丢弃旧代次的加载更多结果，页面释放后慢返回不能继续写入已释放状态。
- 评论列表、评论项和楼层回复弹层的 controller 由 feature 内 controller factory 创建，Widget 不直接拼装 `CommentRepository`。
- 云盘页首次加载会先展示本地账号作用域缓存，后台远程刷新失败时保留已展示列表并挂载错误状态。
- 云盘页首次本地缓存读取失败时会继续尝试远程第一页，缓存层短暂异常不能直接打断云盘入口加载。
- 云盘页面控制器和仓库会拒绝空白用户 id，避免空账号读取或写入云盘缓存，也不会触发远程云盘请求。

- 图片展示缓存和临时目录清理已在分析和删除前归一普通路径和合法 `file://` 路径，避免 legacy 资源索引路径形态不同导致误统计或误删。
- 本地导入、资源索引、播放源解析、播放源预取、播放恢复队列、图片缓存、封面路径选择、封面取色和缓存清理已复用 `LocalFilePathNormalizer`，避免不同入口对 `file://`、远程 URL 和 Windows 盘符路径产生不同判断。
- 音乐数据仓库和播放队列 mapper 输出本地音频、封面和歌词路径前会统一归一，历史 `file://` 或带 query 的索引路径不会继续扩散到播放模型和展示层。
- 播放队列 `isCached` 只有在正式下载或播放缓存音频资源拥有可归一的本地路径时才置 true，远程 URL 或不安全 `file://` 不再误显示为已缓存。
- 播放队列恢复缓存和 `MediaItem` adapter 已在写出和读取时重新校验 `isCached/cache` 标记，只有非本地导入、媒体类型匹配且本地音频文件仍存在时才保留为已缓存。
- 播放队列从 `MediaItem` extras 读回播放地址时只恢复本地路径，临时远程 URL 不再回流到 active queue 或当前歌曲事实。
- 播放恢复队列缓存写入和读取 `localArtworkPath`、`localLyricsPath` 时会归一化合法本地路径，并丢弃远程 URL 或其它 URI scheme。
- 曲目封面预缓存检查已有索引资源时已归一普通路径和合法 `file://` 路径，避免把历史封面误判为缺失后重复下载。
- 远程图片写入本地图片缓存失败时会删除当前 `.download` 临时文件，同一 URL 后续可重新下载而不被失败残留污染。
- 曲目封面预缓存下载失败时会删除当前 `.download` 临时文件，避免失败封面残留污染缓存统计和后续重试。
- 图片取色内存缓存已限制最大条目数，并按最近使用顺序淘汰，避免切换大量封面后无界增长。
- 图片取色持久缓存清理会同步移除访问索引，避免颜色值已删除但轻量视觉缓存元数据继续残留。
- 本地封面取色失败会降级为默认明暗色，不中断页面背景更新，也不把失败结果写入取色缓存。
- 歌词加载、封面取色预热、播放缓存和封面补全统一由当前歌曲副作用调度器执行；同 channel 旧任务可取消，后台失败由装配层记录且不会泄漏到全局异步错误通道。
- 播放状态同步器的 stream 回调、播放位置持久化、当前歌曲持久化、自动续播和播放源错误恢复已统一走播放后台任务错误边界，失败会交给装配层记录，不再泄漏到全局异步错误通道。
- 远程图片 URL 判定已统一到 `ImageUrlNormalizer`，HTTP(S) scheme 大小写不敏感，网易云图片尺寸 `param` 会被移除但其它 query 签名参数继续保留。
- 远程图片 URL 只有尺寸 `param` 一个 query 时也会清空尺寸 query，避免低尺寸封面进入本地图片缓存。
- 曲目封面预缓存下载已复用规整后的远程图片 URL，避免下载低尺寸封面或把尺寸参数写入下载请求。
- 本地图片缓存下载、pending 合并和路径缓存已复用规整后的远程图片 URL，避免同一封面不同尺寸参数重复下载。
- 播放封面展示、歌单封面取色和通用图片 UI 门面已共享启动链路注入的 `LocalImageCacheRepository`，避免展示层各自创建图片缓存仓库。
- 播放队列 mapper 已保持 `artworkUrl` 只承载规整后的远程封面，索引本地封面只写入 `localArtworkPath`，避免本地封面路径覆盖远程兜底。
- 播放封面取色、预热和本地图片预取已优先使用队列项已有本地封面路径，本地路径无效时才回退远程封面 URL。
- 播放页和 mini player 封面展示已改为统一封面路径选择工具处理，本地路径可用时不再先走远程封面 URL。

- 播放队列从 `MediaItem` extras 读回封面时会把本地 `image` 迁移到 `localArtworkPath`，不会把本地封面路径污染到远程 `artworkUrl` 字段。

## 4. 登录和账号

目标边界：

- SDK 管 cookie、token refresh、request session。
- App 管当前用户、账号作用域、本地缓存归属。

需要梳理：

- 冷启动恢复顺序。
- cookie 失效和刷新失败。
- 登出和切换账号。
- `user_*` 表和刷新 marker 的 `user_id` 隔离。
- 账号切换时，所有异步本地/远程刷新结果必须在落状态前复核当前 `userId`。

本地缓存登录必须同时具备登录标记和可解析、`userId` 非空的 session；损坏、匿名或缺少登录标记的孤立 session 不能把 App 带入首页，也不能被用户 session 控制器恢复成当前用户。
登录页启动和后台校验只按完整 cached session 分支，孤立登录标记不能触发账号恢复；SDK cookie 仍只作为接口请求会话。
用户主动注销时，App 当前用户、账号作用域和本地登录标记必须立即清理；远程 logout 失败不能阻塞本地退出。
清理完成后必须通过 Auth 展示副作用回到登录页，避免用户停留在已登录壳层继续触发账号作用域页面。
用户 session 持久化必须串行执行并带代次保护，慢保存、注销清理和新账号写入不能互相覆盖；后台保存失败只能等待下一次 session 变化重试。
用户 session 持久化队列已由架构测试固定，旧账号保存排队期间发生新账号写入时，中间过期保存不会再进入本地 session 缓存。
Auth 缓存恢复和二维码登录流程必须按流程代次写入当前用户；注销、刷新二维码或控制器释放后，旧账号加载、旧二维码创建和旧轮询响应都不能继续落状态。
Auth 账号加载、二维码轮询和后台校验的代次保护已由架构测试固定，跨账号返回的旧结果必须在落状态前复核当前流程和当前用户。
Auth 控制器只能依赖当前 session 的窄读写能力，不能直接查找或命名全局用户会话控制器。

## 5. 缓存策略

需要统一：

```text
歌曲详情      Drift 长期保存，远程刷新补全
歌单          元信息和曲目关系分开缓存
歌词          缺失拉取，成功后长期保存
封面          文件缓存和取色缓存分层
播放 URL      短 TTL 或内存缓存
喜欢歌曲      账号作用域
用户歌单      账号作用域
云盘          账号作用域
评论          短期缓存或后续建专表
搜索热词      短 TTL
```

账号作用域缓存读取失败不能直接阻断核心页面远程加载；已有可见数据时远程失败保留旧数据，没有本地数据时应继续尝试远程第一页。

## 6. 播放和下载

播放方向：

- 本地资源优先。
- 远程 URL 过期后刷新。
- 远程 URL 保留 query 签名参数，只在本地文件路径判断时裁剪 query。
- 播放恢复队列缓存已收紧为只保留明确本地路径，`file://` 会归一为本地文件路径，远程 URL 和其它 URI scheme 不进入恢复 session。
- 播放 URL 短 TTL、并发请求合并、本地资源优先重查和 LRU 淘汰收口在 `PlaybackUrlCacheCoordinator`，避免 `MusicDataRepository` 继续堆播放缓存状态。
- 远程替换失败时强制刷新 URL；高音质仍失败时降级到普通音质。
- SDK 不内置 unblockmusic-utils 替代链路；`song_url_v1` 的 `unblock=true` 会降级到普通播放 URL 请求，source order 不参与普通播放 URL 请求，App 播放链路负责失败后的刷新和音质降级。
- 本地导入文件播放时不在队列模型或播放源替换阶段追加播放缓存展示标记，避免把用户导入资源误当成播放缓存；正式下载和播放缓存资源仍可展示为已缓存。
- 下一首预取不阻塞当前播放。
- 相邻歌曲预取只在当前歌曲 confirmed 后后台启动，不能拖慢当前切歌和控制反馈。
- 恢复状态只保存 restore/session 语义。

下载方向：

- `download_tasks` 描述下载过程。
- `local_resource_entries` 描述最终可用资源。
- 播放只信任本地资源索引，不直接信任下载任务状态，也不能把本地曲目的 `sourceId` 当作隐式本地文件路径。
- 取消下载不能提前消费取消标记；已取消任务即使文件写入完成，也不能登记为本地资源。
- 正式下载在保存本地资源索引前会再次复核取消状态，取消发生在音频落盘后的封面或歌词处理阶段时，也不会登记 `local_resource_entries`。
- 本地资源索引只能代表真实存在的文件，读取或播放入口二次确认发现文件缺失时需要清理索引并回退到远程解析。
- 资源索引写入按本地导入、正式下载、播放缓存维护优先级。
- 正式下载遇到已有本地导入音频资源时会保留 `localImport` 归属，不会提升或覆盖为 `managedDownload`。
- 下载资源写入器会拒绝把 `localImport` bundle 提升成 `managedDownload`，资源归属保护不只依赖上层工作流。
- 批量下载规划会跳过真实存在的本地导入和正式下载音频资源，缺失文件的索引仍会重新进入下载路径。
- 已有播放缓存被正式下载命中时，只提升本地资源索引为正式下载事实并清理任务状态，不重新下载音频文件。
- 正式下载和播放临时缓存判断已有音频资源前已归一普通路径和合法 `file://` 路径，避免历史索引资源被误判为缺失后重复下载。
- 播放缓存清理只删除播放缓存来源资源，不能误删同一曲目的正式下载、本地导入、封面或歌词资源。

## 7. UI 方向

定位：自用、简约、纯粹听歌。

核心页面：

- 首页：最近播放、每日推荐、常用歌单。
- 资料库：喜欢、歌单、本地音乐、下载、云盘。
- 搜索：搜索框、热搜、历史、结果。
- 播放页：封面、歌曲信息、进度、歌词、队列、音质。
- 歌单页：封面、标题、操作区、歌曲列表。
- 设置页：账号、音质、缓存、下载、外观、调试。

避免：

- 信息流、复杂社交、运营 banner 成为主入口。
- 卡片套卡片。
- 过度模糊或取色影响可读性。
- 动效影响播放控制响应。

## 8. 性能指标

性能指标以 `AppPerformanceMetrics` 为准：

```text
key                       指标                 目标     事件
cold_start_interactive    冷启动到可交互       1500ms   app.startup.interactive
cached_playlist_open      打开已有缓存歌单     500ms    page.loadInitial.total
track_switch              切歌响应             300ms    switch.total
search_first_results      搜索首批结果         800ms    search.firstResults.total
mini_player_feedback      mini player 控制反馈 120ms    miniPlayer.feedback.total
```

所有 `elapsedMetric` 输出必须带上 `metric`、`target`、`status=ok/slow` 和剩余或超出预算，避免性能目标只停留在文档表格里。

搜索结果按歌曲、歌单、专辑、歌手分类独立发布，首个分类完成后立即更新状态，不能被最慢分类阻塞；远程搜索失败但已有本地结果时，先保留本地结果作为降级展示。

每次修改后的固定验证门槛：

```bash
flutter analyze
按修改范围运行目标测试
git diff --check
```

这台开发机后续默认不执行 Android APK 构建和 adb 安装；目标测试也按改动范围选择，避免为小改动触发全量耗时验证。只有用户单独要求或切换到更合适的构建环境时才补跑完整构建安装。

mini player 播放/暂停命令在播放 UI 命令服务层合并并发请求，避免快速重复点击打出多次底层播放命令。

mini player 播放/暂停反馈指标会在成功和失败路径都记录 `action` 与 `result`，底层命令异常不能让反馈耗时样本丢失。

播放页喜欢命令在播放控制器边界按曲目合并并发请求，避免快速重复点击造成喜欢态抖动和重复队列同步。

## 9. 建议 goal 顺序

1. 收紧 SDK 与 App 边界。
2. 明确登录、cookie、账号状态所有权。
3. 统一核心缓存策略。
4. 重构播放源解析和 URL 缓存。
5. 整理下载和本地资源索引。
6. 收紧 bootstrap 装配边界。
7. 拆分 Drift 本地数据层。
8. 梳理首页、资料库、搜索、播放页、mini player、歌单页。
9. 优化视觉系统和核心动效。
10. 建立启动、歌单打开、搜索、切歌等性能指标。

## 10. 验收命令

本机验证门槛：

```bash
flutter analyze
flutter test
git diff --check
```

这台机器性能和磁盘余量都不适合作为每次修改后的 Android 构建机，后续开发不默认编译和安装。

架构改动可先补跑：

```bash
flutter test test/architecture
```

但架构测试不能替代本机验证门槛。
