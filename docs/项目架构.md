# 项目架构

## 1. 项目定位

本项目是自用的第三方网易云音乐播放器，目标是：

- 本地优先。
- 播放稳定。
- 响应快。
- UI 简约。
- 能持续跟随上游网易云接口变化。

当前不推倒重来，也不追求形式化分层。普通页面默认保持：

```text
Page / Widget -> Controller -> Repository -> MusicDataRepository / AppStorage
```

## 2. 目录职责

```text
lib/app        启动、依赖装配、路由
lib/core       纯 Dart 基础实体、状态、工具
lib/data       本地存储、远程数据源、数据聚合
lib/features   controller、repository、复杂 feature service
lib/ui         页面、widget、主题、展示服务
packages/netease_music_api  网易云接口 Dart SDK
```

中小项目通用架构参考见：

```text
docs/中小项目架构规范.md
```

本项目不机械照搬参考目录，只保证职责边界一致。

## 3. 硬边界

- `core` 和 `data` 不依赖 Flutter UI、GetX controller、页面和 Widget。
- `core/entities` 保持纯 Dart，不 import DAO、SDK DTO、Flutter 或 GetX。
- Repository 不依赖 UI、`BuildContext`、Widget、Toast、Dialog、GetX 容器。
- UI 和 controller 不直接调用 `NeteaseMusicApi`。
- `app/bootstrap` 只编排 SDK 创建和数据装配，不能直接命名 SDK 类型或发网易云业务请求。
- 网易云协议细节只留在 `data/music_data/sources/netease` 和 `packages/netease_music_api`。
- `audio_service`、`just_audio`、`MediaItem` 只留在 playback 内部适配层。
- `CacheBox.instance` 不继续扩散到业务层。

## 4. 数据所有权

### Drift

Drift 是业务事实来源，保存长期可复用数据：

- 歌曲、歌词、专辑、歌手。
- 歌单和歌单歌曲关系。
- 用户资料、用户歌单、喜欢歌曲、订阅状态。
- 播放恢复、下载任务、本地资源索引。
- 最近播放历史。
- 通用短期 cache marker。

### Hive

Hive 只保存轻量数据：

- 登录态。
- 设置项。
- session。
- 图片取色等视觉缓存。

登录标记、session、设置项和视觉缓存通过窄 key-value 边界访问 Hive，业务和 UI 不直接读取 `CacheBox.instance`。
`CacheBox.instance` 只允许留在 `HiveKeyValueStore` 这一层适配器里。
本地登录恢复必须同时具备登录标记和 App 保存的有效用户 session；SDK cookie 只用于接口请求和刷新，不能单独决定当前用户。
主动注销时本地用户 session 和登录标记优先清理，远程 logout 只作为 SDK 会话清理尝试。
注销后的登录页跳转统一由 Auth 展示副作用触发，UI 不直接操作登录路由。
图片取色内存缓存必须有上限，并按最近使用顺序淘汰，避免切换大量封面后无界增长。
图片取色持久缓存清理必须同步清理访问索引，避免视觉缓存元数据长期残留。

禁止把喜欢歌曲、用户歌单、歌单详情、云盘列表、播放恢复队列等业务事实写入 Hive。
`app_cache_keys.dart` 不保留旧的喜欢歌曲、用户歌单、推荐歌曲、下载任务、本地资源索引或播放恢复快照 key。
搜索热词、探索分类和评论等短期缓存 key 归属 Drift `AppCacheDataSource` 边界，不放在 `data/app_storage`。

### AppCache

`app_cache_entries` 只用于：

- TTL marker。
- 刷新时间。
- 热搜、探索页、评论页等短期 payload。

不作为长期业务事实来源。

## 5. ID 规则

- 新写入实体 ID 使用 `sourceKey:sourceId`。
- 网易云实体使用 `netease:<id>`。
- 旧纯数字 ID 只允许作为兼容输入。
- 关系表优先关联规范化实体 ID。

## 6. 本地优先流程

页面默认流程：

1. 先读 Drift 或本地资源索引。
2. 有本地数据时立即展示。
3. 需要刷新时再请求网络。
4. 网络成功后写回本地。
5. 网络失败且本地有数据时继续展示旧数据。

手动刷新可以绕过 TTL。
账号资料库刷新采用快照式更新：喜欢歌曲 id 和用户歌单远程都成功后才替换当前可见状态；失败且已有本地或可见数据时继续展示旧数据。
页面存在可切换上下文时，分类、榜单、账号、喜欢歌曲 id 等上下文必须在异步结果落状态前复核，旧请求慢返回不能覆盖当前选择。

## 7. 播放链路

页面只通过 `PlayerController` 发播放命令。播放内部负责：

- 播放队列。
- 播放源解析。
- 后台播放。
- 通知栏同步。
- 恢复状态。
- 下载资源协调。

播放源优先级：

```text
本地导入文件 -> 离线下载文件 -> 可用缓存文件 -> 远程播放 URL
```

本地资源索引写入必须维护该优先级；低优先级播放缓存不能覆盖仍存在的本地导入或正式下载资源。
本地导入入口只允许真实存在的本地音频文件生成曲目；远程 URL、空路径和不存在的音频文件不能写入本地曲库。
本地导入会把同名封面和歌词作为本地资源登记；副资源扩展名匹配不区分大小写，但写入索引时保留真实文件路径。
本地导入文件播放时只作为本地文件事实处理，不能在队列模型或播放源替换阶段追加播放缓存展示标记；正式下载和播放缓存资源仍可展示为已缓存。
本地导入、资源索引、播放源解析、播放源预取、播放恢复队列、图片缓存、封面路径选择、封面取色和缓存清理必须复用同一套本地路径归一规则。
播放队列和播放 URL 解析只能从 `local_resource_entries` 读取本地音频路径，不能把本地曲目的 `sourceId` 当作隐式文件路径。
正式下载如果命中已有播放缓存文件，只能提升 `local_resource_entries` 中的资源事实并清理 `download_tasks`，不能重新下载或只凭任务状态判断已完成。
播放缓存清理只删除 `origin=playbackCache` 的资源文件和索引，不能按整首歌删除其他来源资源。
播放缓存写入失败时必须删除未登记到资源索引的音频、封面和歌词文件，避免留下不可追踪的孤儿缓存。
播放入口二次确认本地资源文件缺失时，必须同步删除对应 `local_resource_entries` 索引，再回退到远程解析或明确失败。
封面预缓存清理必须以本地资源索引判断可删除文件；被正式下载、本地导入或其他保留索引引用的封面文件不能被设置页缓存清理删除。
播放缓存和封面预缓存清理在判断保留引用前必须归一普通本地路径、合法 `file://` 和 Windows 盘符路径，避免历史索引路径形态不同导致误删共享文件。
远程播放 URL 必须保留 query 签名参数；只有本地文件路径判断可以裁剪 query。
播放恢复队列缓存只保留普通本地路径、合法 `file://` 路径和 Windows 盘符路径；远程 URL 和其它 URI scheme 必须丢弃，避免把临时播放地址持久化为 session 事实。
播放 URL 和播放源预取缓存必须按音质偏好隔离，避免普通音质和高音质 URL 串用。
播放 URL 内存缓存必须有上限，并按最近使用顺序淘汰，避免长时间播放大量不同歌曲后无界增长。
播放 URL 短 TTL、并发请求合并、本地资源优先重查和 LRU 淘汰由 `PlaybackUrlCacheCoordinator` 负责，`MusicDataRepository` 只负责编排本地资源索引、本地扫描来源和远程来源。
播放源预取缓存必须有上限，命中时刷新最近使用顺序，过期或本地文件不可用时必须剔除后重新解析。
相邻歌曲播放源预取只能在当前切歌确认后以后台任务启动，不能阻塞当前歌曲的 confirmed 状态和控制反馈。
远程 URL 替换失败时必须绕过短 TTL 缓存重新取地址，再尝试恢复播放。
高音质远程 URL 替换仍失败时，播放链路必须继续降级到普通音质 URL。
最近播放历史只记录底层已确认播放的曲目 id 和播放时间，不能混入播放恢复队列、UI 选择态或 SDK raw 数据。
首页“最近播放”只能从本地播放历史读取并映射为播放队列项，不能用当前播放队列冒充历史。
最近播放入口只能由播放历史写入完成事件刷新，不能监听 UI 当前选择态来推断播放历史。
首页快速入口优先保留“继续播放”，有当前歌曲时直接打开播放面板并在暂停时恢复播放；没有当前歌曲时入口保持禁用。
首页歌单区只展示少量“常用歌单”，目前按账号作用域用户歌单顺序截取前 8 个；完整歌单列表不进入首页主路径。
首页资料库入口先保持轻量，只承接本地音乐、已下载和云盘；喜欢歌曲和常用歌单仍保留在个人页主路径。
个人页的最近播放横条和资料库快捷入口已下沉到页面局部 widgets；个人页主文件只负责编排首页、方屏首页和推荐歌单入口。
歌单页的头图操作区、歌曲骨架列表和底部状态提示已下沉到页面局部 widgets；页面主文件继续保留加载、缓存、播放和订阅状态机。
底部播放面板的收起态 mini player、背景层、PageView 边缘遮罩、正在播放页、专辑/歌手元信息区、页面指示器、封面图片解析、分页滚动和封面卡片细节已下沉到 shell 播放面板局部 widgets；主面板继续负责编排 PageView 页面顺序和封面层入口。
顶部搜索面板的背景层、底部 tab/searchbar/键盘占位、内容区组合、内容卡片、热词列表、搜索结果状态列表和歌单/专辑/歌手结果条目已下沉到 shell 搜索局部 widgets；主文件继续保留搜索触发、面板开合和路由打开编排。
设置页的分组容器和跳转/开关条目已下沉到页面局部 widget，本地扫描权限、默认目录解析和导入动作已下沉到 `LocalMediaScanController`；主文件继续保留账号入口、路由跳转和扫描 UI 反馈。

## 8. 当前风险

- bootstrap 已按 UI、route、SDK、storage、数据源、repository、数据装配、播放、展示适配器、feature/controller 拆分，后续只保留装配职责。
- Drift 表定义已按业务域拆到 schema part，迁移和索引也已拆出主数据库入口。
- 用户作用域 data source 已拆成窄接口，窄 Drift 实现只保留 DAO 委托。
- `PlaybackQueueItem` 已移除 adapter 和缓存 JSON 职责，`albumId`、`artistIds`、`sourceType`、`localLyricsPath`、`availability` 已字段化；UI 必须使用显式字段，动态 `metadata` 键访问只允许留在 mapper、MediaItem adapter 和缓存 codec 边界。
- UI 大页面拆分边界已阶段性收敛，后续风险转向真实设备上的封面动效、可读性和核心路径交互回归验证。
