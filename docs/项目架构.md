# 项目架构

## 1. 项目定位

本项目是自用的第三方网易云音乐播放器，目标是：

- 本地优先。
- 播放稳定。
- 响应快。
- UI 简约。
- 能持续跟随上游网易云接口变化。

当前不推倒重来，也不追求形式化分层。普通页面默认保持：

```text
Page / Widget -> Controller -> Repository -> MusicDataRepository / AppStorage
```

## 2. 目录职责

```text
lib/app        启动、依赖装配、路由
lib/core       纯 Dart 基础实体、状态、工具
lib/data       本地存储、远程数据源、数据聚合
lib/features   controller、repository、复杂 feature service
lib/ui         页面、widget、主题、展示服务
packages/netease_music_api  网易云接口 Dart SDK
```

中小项目通用架构参考见：

```text
docs/中小项目架构规范.md
```

本项目不机械照搬参考目录，只保证职责边界一致。

## 3. 硬边界

- `core` 和 `data` 不依赖 Flutter UI、GetX controller、页面和 Widget。
- `core/entities` 保持纯 Dart，不 import DAO、SDK DTO、Flutter 或 GetX。
- Repository 不依赖 UI、`BuildContext`、Widget、Toast、Dialog、GetX 容器。
- UI 和 controller 不直接调用 `NeteaseMusicApi`。
- 网易云协议细节只留在 `data/music_data/sources/netease` 和 `packages/netease_music_api`。
- `audio_service`、`just_audio`、`MediaItem` 只留在 playback 内部适配层。
- `CacheBox.instance` 不继续扩散到业务层。

## 4. 数据所有权

### Drift

Drift 是业务事实来源，保存长期可复用数据：

- 歌曲、歌词、专辑、歌手。
- 歌单和歌单歌曲关系。
- 用户资料、用户歌单、喜欢歌曲、订阅状态。
- 播放恢复、下载任务、本地资源索引。
- 通用短期 cache marker。

### Hive

Hive 只保存轻量数据：

- 登录态。
- 设置项。
- session。
- 图片取色等视觉缓存。

设置项和视觉缓存通过窄 key-value 边界访问 Hive，业务和 UI 不直接读取 `CacheBox.instance`。

禁止把喜欢歌曲、用户歌单、歌单详情、云盘列表、播放恢复队列等业务事实写入 Hive。

### AppCache

`app_cache_entries` 只用于：

- TTL marker。
- 刷新时间。
- 热搜、探索页等短期 payload。

不作为长期业务事实来源。

## 5. ID 规则

- 新写入实体 ID 使用 `sourceKey:sourceId`。
- 网易云实体使用 `netease:<id>`。
- 旧纯数字 ID 只允许作为兼容输入。
- 关系表优先关联规范化实体 ID。

## 6. 本地优先流程

页面默认流程：

1. 先读 Drift 或本地资源索引。
2. 有本地数据时立即展示。
3. 需要刷新时再请求网络。
4. 网络成功后写回本地。
5. 网络失败且本地有数据时继续展示旧数据。

手动刷新可以绕过 TTL。

## 7. 播放链路

页面只通过 `PlayerController` 发播放命令。播放内部负责：

- 播放队列。
- 播放源解析。
- 后台播放。
- 通知栏同步。
- 恢复状态。
- 下载资源协调。

播放源优先级：

```text
本地导入文件 -> 离线下载文件 -> 可用缓存文件 -> 远程播放 URL
```
