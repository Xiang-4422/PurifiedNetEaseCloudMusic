# 项目架构

## 1. 项目定位

本项目是自用的第三方网易云音乐播放器，目标是：

- 本地优先。
- 播放稳定。
- 响应快。
- UI 简约。
- 能持续跟随上游网易云接口变化。

当前不推倒重来，也不追求形式化分层。普通页面默认保持：

```text
Page / Widget -> Controller -> Repository -> MusicDataRepository / AppStorage
```

## 2. 目录职责

```text
lib/app        启动、依赖装配、路由
lib/core       纯 Dart 基础实体、状态、工具
lib/data       本地存储、远程数据源、数据聚合
lib/features   controller、repository、复杂 feature service
lib/ui         页面、widget、主题、展示服务
packages/netease_music_api  网易云接口 Dart SDK
```

中小项目通用架构参考见：

```text
docs/中小项目架构规范.md
```

本项目不机械照搬参考目录，只保证职责边界一致。

## 3. 硬边界

- `core` 和 `data` 不依赖 Flutter UI、GetX controller、页面和 Widget。
- `core/entities` 保持纯 Dart，不 import DAO、SDK DTO、Flutter 或 GetX。
- Repository 不依赖 UI、`BuildContext`、Widget、Toast、Dialog、GetX 容器。
- UI 和 controller 不直接调用 `NeteaseMusicApi`。
- `app/bootstrap` 只编排启动装配；只有 `sdk_bootstrap`、`data_bootstrap`、`data_source_bootstrap` 能携带 `NeteaseMusicApi` 门面，其他 bootstrap 文件不能命名 SDK 类型或发网易云业务请求。
- `app_bootstrap.dart` 只作为最外层编排入口，import 只能指向子 bootstrap、Flutter binding 和 GetX，不能直接导入 data、features、ui 或 SDK 业务文件。
- `lib/app.dart` 只承接根路由、主题、滚动行为和依赖装配入口；登录失效 Toast/跳转副作用监听下沉到 UI widget，App 根不直接导入 feature、data 或展示服务细节。
- 网易云协议细节只留在 `data/music_data/sources/netease` 和 `packages/netease_music_api`。
- Feature repository 通过 `data/music_data/music_remote_data_sources.dart` 的中性远程契约接入远程能力，不能直接 import `data/music_data/sources/netease/remote` 的具体实现。
- `audio_service`、`just_audio`、`MediaItem` 只留在 playback 内部适配层。
- 播放队列转换为 `MediaItem` 时，本地封面 `artUri` 必须复用本地路径归一规则；远程 URL 或非法本地路径不能伪装成本地文件 URI。
- 播放队列转换为 `MediaItem` 时，extras 的 `image` 字段必须优先写入可用本地封面路径；本地路径无效时才回退远程封面 URL。
- `CacheBox.instance` 不继续扩散到业务层。

## 4. 数据所有权

### Drift

Drift 是业务事实来源，保存长期可复用数据：

- 歌曲、歌词、专辑、歌手。
- 歌单和歌单歌曲关系。
- 用户资料、用户歌单、喜欢歌曲、订阅状态。
- 播放恢复、下载任务、本地资源索引。
- 最近播放历史。
- 通用短期 cache marker。

### Hive

Hive 只保存轻量数据：

- 登录态。
- 设置项。
- session。
- 图片取色等视觉缓存。

登录标记、session、设置项和视觉缓存通过窄 key-value 边界访问 Hive，业务和 UI 不直接读取 `CacheBox.instance`。
`CacheBox.instance` 只允许留在 `HiveKeyValueStore` 这一层适配器里。
本地登录恢复必须同时具备登录标记和 App 保存的有效用户 session；路由、登录流程和 `UserSessionController` 复用同一条恢复判断，孤立登录标记、孤立 session 或 SDK cookie 都不能单独决定当前用户。
主动注销时本地用户 session 和登录标记优先清理，远程 logout 只作为 SDK 会话清理尝试。
注销后的登录页跳转统一由 Auth 展示副作用触发，UI 不直接操作登录路由。
登录流程通过 `AuthSessionAccess` 读写当前 App session，`AuthController` 不直接命名全局 `UserSessionController`。
用户资料库通过 `UserLibrarySessionAccess` 读取当前账号、等待 session 缓存并监听账号变化，真实 `UserSessionController` 只允许在 bootstrap 装配层绑定。
图片取色内存缓存必须有上限，并按最近使用顺序淘汰，避免切换大量封面后无界增长。
图片取色持久缓存清理必须同步清理访问索引，避免视觉缓存元数据长期残留。
封面取色遇到本地图片缺失、损坏或无法解码时必须降级为默认明暗色，不能中断页面背景更新或把失败结果写入缓存。

禁止把喜欢歌曲、用户歌单、歌单详情、云盘列表、播放恢复队列等业务事实写入 Hive。
`app_cache_keys.dart` 不保留旧的喜欢歌曲、用户歌单、推荐歌曲、下载任务、本地资源索引或播放恢复快照 key。
搜索热词、探索分类和评论等短期缓存 key 归属 Drift `AppCacheDataSource` 边界，不放在 `data/app_storage`。

### AppCache

`app_cache_entries` 只用于：

- TTL marker。
- 刷新时间。
- 热搜、探索页、评论页等短期 payload。

不作为长期业务事实来源。

## 5. ID 规则

- 新写入实体 ID 使用 `sourceKey:sourceId`。
- 网易云实体使用 `netease:<id>`。
- 旧纯数字 ID 只允许作为兼容输入。
- 关系表优先关联规范化实体 ID。

## 6. 本地优先流程

页面默认流程：

1. 先读 Drift 或本地资源索引。
2. 有本地数据时立即展示。
3. 需要刷新时再请求网络。
4. 网络成功后写回本地。
5. 网络失败且本地有数据时继续展示旧数据。

手动刷新可以绕过 TTL。
账号资料库刷新采用快照式更新：喜欢歌曲 id 和用户歌单远程都成功后才替换当前可见状态；失败且已有本地或可见数据时继续展示旧数据。
用户资料库 controller 必须先规范化当前用户 id，空白账号不能被视为已有本地资料库数据；仓库继续拒绝空白用户 id，不能读取或写入资料、喜欢歌曲、歌单、推荐或 FM 的账号作用域缓存，也不能触发对应远程请求。
用户资料库中的喜欢歌曲列表保留网易云返回或本地缓存的原始顺序，用于“我喜欢的音乐”队列；跨页面喜欢态上下文只能使用 `likedSongIdSnapshot` 或 `normalizeLikedSongIds` 生成的去重排序快照。
页面存在可切换上下文时，分类、榜单、账号、喜欢歌曲 id 等上下文必须在异步结果落状态前复核，旧请求慢返回不能覆盖当前选择。
专辑、歌手等音乐详情页的本地优先控制流归入 `features/music_detail` 和 `ui/pages/music_detail`：feature 层处理本地缓存容错和喜欢歌曲快照，播放 application 层统一补齐曲目队列资源状态，UI 层只复用详情页状态机并渲染具体页面结构。

## 7. 播放链路

页面只通过 `PlayerController` 发播放命令。播放内部负责：

- 播放队列。
- 播放源解析。
- 后台播放。
- 通知栏同步。
- 恢复状态。
- 下载资源协调。

播放源优先级：

```text
本地导入文件 -> 离线下载文件 -> 可用缓存文件 -> 远程播放 URL
```

本地资源索引写入必须维护该优先级；低优先级播放缓存不能覆盖仍存在的本地导入或正式下载资源。
本地导入入口只允许真实存在的本地音频文件生成曲目；远程 URL、空路径和不存在的音频文件不能写入本地曲库。
本地导入会把同名封面和歌词作为本地资源登记；副资源扩展名匹配不区分大小写，但写入索引时保留真实文件路径。
本地导入文件播放时只作为本地文件事实处理，不能在队列模型或播放源替换阶段追加播放缓存展示标记；正式下载和播放缓存资源仍可展示为已缓存。
本地导入、资源索引、播放源解析、播放源预取、播放恢复队列、图片缓存、封面路径选择、封面取色缓存 key 和缓存清理必须复用同一套本地路径归一规则。
本地图片展示缓存仓库由 `repository_bootstrap` 统一创建和注册，播放封面展示、歌单封面取色和通用图片 UI 门面只能接收该实例，不能在 feature 或 UI 内部自行创建缓存仓库。
音乐数据仓库返回本地资源路径、播放队列 mapper 写入 `playbackUrl`、`localArtworkPath` 和 `localLyricsPath` 前都必须输出归一后的普通本地路径，不能把历史 `file://` 或带 query 的索引路径继续传给播放模型。
播放队列 mapper 写入 `PlaybackQueueItem.id` 前必须规范化曲目 id，并跳过空白 id 的曲目或节目；路径类 `sourceId` 不在这里做路径外的裁剪。
`MediaItem` adapter 写入和读回队列项时必须规范化 `PlaybackQueueItem.id`，批量转换时空白 id 不能进入底层播放队列或应用 active queue。
`PlaybackQueueService` 写入 original queue、active queue、漫游追加队列和恢复队列前必须再次规范化 `PlaybackQueueItem.id`，空白 id 不能进入队列事实或 confirmed item 匹配。
播放队列 `isCached` 只能由可归一为本地路径的正式下载或播放缓存音频资源置 true，不能只凭资源来源字段或远程 URL 显示已缓存。
播放队列写入恢复缓存、写出 `MediaItem` extras 或读取旧 `isCached/cache` 标记时，都必须重新校验非本地导入、媒体类型和本地文件存在性，不能把持久化标记当作资源事实。
播放队列从 `MediaItem` extras 读回 `url` 时只恢复本地路径；远程播放 URL 只能作为临时解析结果使用，不能回流到 active queue 或当前歌曲事实。
播放封面取色、预热和本地图片预取必须优先使用队列项已有本地封面路径；本地路径无效时才回退远程封面 URL。
`PlaybackQueueItem.artworkUrl` 只承载规整后的远程封面，`localArtworkPath` 单独承载索引本地封面；展示层通过统一封面路径选择工具做本地优先，不能靠混用字段获得本地封面。
播放队列缺封面时必须通过播放仓库读取可用封面来源，由音乐数据仓库确认本地封面文件仍存在并清理失效索引；播放展示层不能直接信任 `TrackResourceBundle.artwork.path`。
图片列表项需要缩略图但没有可用图片地址时必须保留统一占位尺寸；只有调用方明确不需要缩略图时才传 `null`，避免列表标题列在加载、缺图和有图状态之间横向跳动。
播放页和 mini player 的封面展示必须通过统一封面路径选择工具处理本地优先，不能在 Widget 内手写 `artworkUrl ?? localArtworkPath`。
首页快捷卡、最近播放横条、每日推荐头图和底部播放 header 只要展示播放队列项封面，也必须通过统一封面路径选择工具处理本地优先。
最近播放横条缺封面时也必须走统一图片组件的空图占位，不能在局部 Widget 手写另一套封面占位分支。
通用歌曲列表项展示播放队列项封面时必须复用同一工具，确保歌单、搜索、云盘、今日推荐等列表默认本地封面优先。
页面级封面没有明确图片时，统一封面路径选择工具可使用候选曲目的远程 HTTP(S) 封面兜底，但仍必须交给统一图片组件进入本地图片缓存，不能绕过缓存直接展示网络图。
歌词资源索引读取文件前必须归一普通路径和合法 `file://` 路径，避免通过存在性检查后又按原始 URI 读取失败。
播放队列和播放 URL 解析只能从 `local_resource_entries` 读取本地音频路径，不能把本地曲目的 `sourceId` 当作隐式文件路径。
正式下载如果命中已有播放缓存文件，只能提升 `local_resource_entries` 中的资源事实并清理 `download_tasks`，不能重新下载或只凭任务状态判断已完成。
数据库里残留的 `completed` 下载任务不能被当成已下载事实；没有可用 `managedDownload` 音频资源索引时必须重新进入正式下载流程。
批量下载规划只有在 `managedDownload` 音频资源索引指向真实存在的本地文件时才能跳过下载；索引文件缺失时必须重新入队，由正式下载流程覆盖资源事实。
下载资源写入器在写入任何资源索引前，必须先规范化曲目 id 和资源路径；空白曲目 id、不可归一路径或不存在的主音频文件不能写入 `local_resource_entries`。清理任务或保存封面、歌词等附属资源前，必须确认主音频索引来源符合预期、路径可归一且文件真实存在，不能只凭 `local_resource_entries` 的来源字段判断下载完成。
下载仓库和批量下载规划必须在任务入口先规范化曲目 id，再拒绝空白曲目 id；带空格的 id 不能形成重复 `download_tasks`，也不能绕过本地资源查询或 active task 判断。
下载任务 DAO 写入、读取和删除 `download_tasks` 前也必须规范化曲目 id 并拒绝空白 id，持久化边界不能只依赖上层 store。
本地资源索引仓库必须在读取、写入、访问时间刷新和删除入口拒绝空白曲目 id，空白 id 不能写入 `local_resource_entries`。
正式下载和播放临时缓存判断已有音频资源前必须归一普通路径和合法 `file://` 路径，避免历史索引资源被误判为缺失后重复下载。
本地歌曲列表读取到资源索引但找不到对应曲目事实时必须清理该曲目的资源索引，避免孤儿索引长期保护已不可展示的文件。
播放缓存清理只删除 `origin=playbackCache` 的资源文件和索引，不能按整首歌删除其他来源资源。
播放缓存写入失败时必须删除未登记到资源索引的音频、封面和歌词文件，避免留下不可追踪的孤儿缓存。
播放入口二次确认本地资源文件缺失时，必须同步删除对应 `local_resource_entries` 索引，再回退到远程解析或明确失败。
封面预缓存清理必须以本地资源索引判断可删除文件；被正式下载、本地导入或其他保留索引引用的封面文件不能被设置页缓存清理删除。
图片展示缓存、临时目录、播放缓存和封面预缓存清理都必须以本地资源索引判断可删除文件；分析页只统计可安全清理的文件，不能把仍被索引引用的文件计入可清理容量。
播放缓存、封面预缓存和通用目录清理在判断保留引用前必须归一普通本地路径、合法 `file://` 和 Windows 盘符路径，避免历史索引路径形态不同导致误删共享文件；音乐数据仓库删除本地资源时必须通过 `LocalResourceRetentionPolicy` 计算保留路径和删除许可。
曲目封面预缓存检查已有索引资源时必须归一普通路径和合法 `file://` 路径，避免把历史封面误判为缺失后重复下载。
远程播放 URL 必须保留 query 签名参数；只有本地文件路径判断可以裁剪 query。
远程图片 URL 的 HTTP(S) scheme 判定必须大小写不敏感；网易云图片只移除尺寸 `param`，其它 query 签名参数继续保留。
远程图片 URL 只有尺寸 `param` 一个 query 时必须移除整个 query，避免低尺寸封面进入本地缓存 key。
曲目封面预缓存下载必须使用规整后的远程图片 URL，避免下载低尺寸封面或把尺寸参数写入下载请求。
本地图片缓存下载、pending 合并和路径缓存必须使用规整后的远程图片 URL，避免同一封面不同尺寸参数重复下载。
播放恢复队列缓存中的 `playbackUrl`、`localArtworkPath` 和 `localLyricsPath` 只保留普通本地路径、合法 `file://` 路径和 Windows 盘符路径；远程 URL 和其它 URI scheme 必须丢弃，避免把临时播放地址或展示字段持久化为 session 事实。
播放恢复队列缓存写入和读取时必须规范化 `PlaybackQueueItem.id`，空白 id 的队列项不能进入恢复 session。
播放恢复状态读取失败时必须降级为空恢复状态，不能阻断启动恢复；失败结果不能写入内存恢复缓存，后续读取仍可重试真实恢复数据。
播放恢复状态的当前歌曲 id 在写入和读取时都必须规范化；纯空白 id 不能让旧恢复状态被误判为有效 session。
播放 URL 和播放源预取缓存必须按音质偏好隔离，避免普通音质和高音质 URL 串用。
播放 URL 内存缓存必须有上限，并按最近使用顺序淘汰，避免长时间播放大量不同歌曲后无界增长。
播放 URL 短缓存必须先规范化曲目 id，再执行本地资源重查、in-flight 合并和 cache key 查询；带空格的 id 不能形成重复远程 URL 缓存，也不能绕过本地资源命中。
播放 URL 短缓存只缓存带 authority 的 HTTP(S) 地址；无效远程地址不能作为可播放结果返回，也不能写入短 TTL 缓存，下一次仍需重新解析。
空白远程播放地址不能作为可播放结果返回，也不能写入短 TTL 缓存，下一次解析必须重新请求。
远程播放 URL 加载失败不能写入短 TTL 缓存，也不能长期占用 in-flight 状态；下一次解析必须重新请求，成功后才能进入缓存。
播放 URL 命中短缓存、合并远程请求或强制刷新远程地址前，本地资源优先重查失败必须降级为本地未命中，不能阻断已有远程 URL、正在进行的远程解析或刷新请求。
播放 URL 本地资源重查返回空白字符串时必须按本地未命中处理，不能把空白播放地址当成本地资源命中返回。
播放 URL 本地资源重查返回值必须经过本地路径归一；远程 URL、不安全 `file://` 和其它 URI scheme 只能按本地未命中处理。
播放 URL 短 TTL、并发请求合并、本地资源优先重查和 LRU 淘汰由 `PlaybackUrlCacheCoordinator` 负责，`MusicDataRepository` 只负责编排本地资源索引、本地扫描来源和远程来源。
`MusicDataRepository` 必须在曲目详情、批量曲目读取、播放 URL、封面、歌词和资源索引入口先规范化曲目 id，再拒绝空白曲目 id；播放 URL 入口必须以规范化 id 进入本地资源查询、短缓存和远程请求；批量曲目和资源索引读取必须按规范化后的候选 id 去重并保持顺序，避免异常队列项触发无效本地查询、重复缓存或网易云远程请求。
播放 URL 短缓存、播放源解析和播放源预取必须通过 `PlaybackSourceReference` 统一判断 HTTP(S) URL、本地路径、现存本地文件和远程 URL 过期状态，不能在各自模块里重复手写 URL/path 判定。
播放源解析器是进入底层播放器前的最终边界；远程播放源必须再次校验为带 authority 的 HTTP(S) URL，本地路径必须通过统一播放引用边界和文件存在性检查，其他 URI scheme 或畸形远程地址必须返回空播放源。
播放源预取缓存必须先规范化曲目 id 和本地播放路径再生成 key，远程临时播放 URL 不能参与 key；缓存必须有上限，命中时刷新最近使用顺序，过期、本地文件不可用或远程 in-flight 期间本地文件恢复时必须剔除或绕过旧结果后重新解析；缓存写入前必须把可用播放源规整为统一播放引用。
播放源预取远程解析失败或解析为空白、文件缺失、过期或畸形远程 URL 等不可用源时，必须归空且不能写入预取缓存，也不能长期占用 in-flight 状态；下一次解析必须重新请求，成功后才能进入预取缓存。
播放源远程强制刷新 key 只能由规范化曲目 id、来源类型和音质偏好组成，不能包含短效 `playbackUrl` 或本地缓存路径；刷新开始时必须清除同 key 的旧远程缓存，后续并发解析只能复用刷新中的请求，不能继续命中过期 URL。
相邻歌曲播放源预取只能在当前切歌确认后以后台任务启动，不能阻塞当前歌曲的 confirmed 状态和控制反馈。
远程 URL 替换失败时必须绕过短 TTL 缓存重新取地址，再尝试恢复播放。
高音质远程 URL 替换仍失败时，播放链路必须继续降级到普通音质 URL。
最近播放历史只记录底层已确认播放的曲目 id 和播放时间，不能混入播放恢复队列、UI 选择态或 SDK raw 数据。
最近播放历史写入必须先规范化曲目 id，空白 id 不能进入播放历史，也不能触发最近播放刷新通知。
首页“最近播放”只能从本地播放历史读取并映射为播放队列项，不能用当前播放队列冒充历史。
最近播放入口只能由播放历史写入完成事件刷新，不能监听 UI 当前选择态来推断播放历史。
最近播放入口在本地历史加载中收到连续写入通知时必须合并为一次追加刷新，避免首页控制器重复读取和重复 rebuild。
最近播放历史读取或曲目资源映射失败时必须降级为空列表；历史写入失败不能阻断当前歌曲保存，修剪失败不能阻断最近播放刷新。
首页快速入口优先保留“继续播放”，有当前歌曲时直接打开播放面板并在暂停时恢复播放；没有当前歌曲时入口保持禁用。
首页歌单区只展示少量“常用歌单”，目前按账号作用域用户歌单顺序截取前 8 个；完整歌单列表不进入首页主路径。
首页资料库入口保持轻量，统一承接我喜欢、我的歌单、本地音乐、已下载和云盘；我喜欢不再作为独立列表项占用首页主路径。资料库快捷入口栏必须保持固定高度、固定按钮宽度、固定 `itemExtent` 和有界预缓存。
个人页的普通首页布局、方屏首页布局、开始听区块、方屏开始听页片、方屏资料库页片、最近播放横条、常用歌单区、资料库入口区和推荐歌单 sliver 已下沉到页面局部 widgets；方屏独立推荐歌单页已从个人页主文件移出，推荐、资料库、播放和最近播放控制器由首页主体组合层注入，个人页主文件只负责布局模式分发。开始听横向入口只保留“继续播放”和“每日推荐”两个核心动作；入口和最近播放横条必须保持固定高度、固定条目宽度、固定 `itemExtent`、惰性构建和有界预缓存，避免首页滚动时重新测量或一次性构建所有卡片。
首页壳层在 `AppHomePageView` 读取 `HomeShellController`，再通过 `HomeShellScope` 传给路由子树；`AppBodyPageView`、抽屉主屏幕和左侧菜单只接收局部上下文边界，不在子组件内部读取全局容器。
首页主导航只保留听歌相关路径：普通屏为“我的音乐、设置”，方屏额外提供“推荐”；探索、捐赠、宣传、运营和其它非听歌入口不能进入主导航分页。
个人首页需要的推荐、资料库、最近播放和播放 controller 由 `PersonalHomeControllerBundle` 在 `feature_bootstrap` 装配，`AppBodyPageView` 只读取该组合边界并向个人页展开注入，不能直接读取 `UserLibraryController`。
歌单页的滚动内容组合、头图操作区、歌曲骨架列表、歌曲列表和底部状态提示已下沉到页面局部 widgets，展示派生规则已收敛到页面局部状态模型；页面主文件继续保留加载、缓存、播放、取色和订阅状态机，controller 与封面取色服务通过 feature 内 controller factory 创建，底部播放面板打开动作经页面 shell 边界统一提交，页面不直接拼装 `PlaylistRepository`、喜欢歌曲 provider、当前账号上下文或本地图片缓存实现，也不在播放动作里读取全局 shell。歌单仓库读写订阅状态前必须先规范化当前用户 id，空白账号不能读取或写入账号作用域订阅缓存，也不能被判断为“我的歌单”。
歌单页读取本地详情、初始化详情和远程详情前，必须由页面 controller 生成去重排序后的喜欢歌曲 id 快照，再传给 `PlaylistRepository`，避免 liked id 重复或换序影响歌单歌曲喜欢态派生。
底部播放面板的收起态 mini player、背景层、PageView 边缘遮罩、正在播放页、专辑/歌手元信息区、页面指示器、封面图片解析、分页滚动和封面卡片细节已下沉到 shell 播放面板局部 widgets；首页壳层负责取用播放控制器、设置控制器和评论 controller factory，主面板只接收这些边界并编排 PageView 页面顺序和封面层入口，不在主面板内部读取全局容器。
收起态 mini player 接收首页壳层注入的播放控制器和设置控制器，只展示当前播放入口、进度和播放按钮，不在 mini player 内部读取全局容器。

收起态 mini player 展开底部播放面板时必须通过 `MiniPlayerExpandCoordinator` 合并并发打开请求，快速重复点击只能复用当前展开任务并记录合并结果。
底部背景层和 PageView 渐隐遮罩接收主面板注入的设置控制器，只消费播放面板取色，不在视觉组件内部读取全局容器。

底部页面指示器接收主面板注入的播放控制器和设置控制器，只展示播放会话标题和 tab 状态，不在指示器内部读取全局容器。

底部播放队列页接收主面板注入的播放控制器和设置控制器，只展示队列并提交播放意图，不在队列页内部读取全局容器。队列列表必须保持 `ListView.builder` 和有界预缓存，避免长队列影响播放面板打开和切歌反馈。
底部评论页接收主面板注入的播放控制器和设置控制器，只根据当前歌曲展示评论，不在评论页内部读取全局容器。
播放页喜欢按钮必须通过 `PlayerController` 播放边界读取喜欢态并切换喜欢，不能在 Widget 内直接读取 `UserLibraryController.likedSongIds` 或切换用户喜欢状态后手写队列更新；同一曲目的并发喜欢切换必须合并。
播放页音质按钮必须通过 `PlayerController` 播放边界读取和切换播放源偏好，不能在 Widget 内直接读写 `SettingsController.isHighSoundQualityOpen`。

## 8. 当前风险

- bootstrap 已按 UI、route、SDK、storage、数据源、repository、数据装配、播放、展示适配器、feature/controller 拆分，后续只保留装配职责。
- Drift 表定义已按业务域拆到 schema part，迁移和索引也已拆出主数据库入口。
- 用户作用域 data source 已拆成窄接口，窄 Drift 实现只保留 DAO 委托。
- `PlaybackQueueItem` 已移除 adapter 和缓存 JSON 职责，`albumId`、`artistIds`、`sourceType`、`localLyricsPath`、`availability` 已字段化；UI 和播放 mapper 必须使用显式字段，动态 `metadata` 键访问只允许留在 MediaItem adapter 和缓存 codec 边界。
- UI 大页面拆分边界已阶段性收敛，后续风险转向真实设备上的封面动效、可读性和核心路径交互回归验证。
